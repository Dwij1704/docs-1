xpander Open Source SDK is designed to empower developers to build intelligent and reliable AI Agents capable of managing complex, multi-step tasks across diverse systems and platforms without focusing on function calling, schema definition, graph enforcement, and prompt group management.

This documentation provides a comprehensive guide to all the classes, methods, and constants available in the **xpander SDK**. 

With support for leading LLM providers such as OpenAI, Amazon Bedrock, and NVIDIA NIM, the **xpander SDK** seamlessly integrates into your existing systems, simplifying function calling, agent orchestration, and tool management.

# Hello World Example

Let's walk through a complete example of using the xpander SDK with OpenAI's GPT-4 model to create an intelligent agent that can perform tasks.

<CodeGroup>
```python OpenAI Integration
from xpander_sdk import XpanderClient, LLMProvider
from openai import OpenAI
import json

# Load environment variables
from dotenv import load_dotenv
import os
load_dotenv()

# Configure API keys
xpanderAPIKey = os.environ.get("XPANDER_API_KEY","")    # Your xpander API Key
xpanderAgentID = os.environ.get("XPANDER_AGENT_ID", "") # Your Agent ID
openAIKey = os.environ.get("OPENAI_API_KEY", "")        # Your OpenAI API Key

# Initialize clients
openai_client = OpenAI(api_key=openAIKey)
xpander_client = XpanderClient(api_key=xpanderAPIKey)
agent1 = xpander_client.agents.get(agent_id=xpanderAgentID)

# Set up conversation memory
memory = []
memory.append({
    "role": "system", 
    "content": "You are a helpful assistant with function calling and tool access. You are running in a while loop - add ##FINAL ANSWER## when complete."
})
memory.append({
    "role": "user", 
    "content": "Get news about qwen2.5-coder"
})

# Main interaction loop
number_of_calls = 1
while True:
    # Track conversation steps
    memory.append({"role": "assistant", "content": f'Step number: {number_of_calls}'})
    
    # Get LLM response
    llm_response = openai_client.chat.completions.create(
        model="gpt-4",
        messages=memory,
        tools=agent1.get_tools()    
    )
    memory.append(llm_response.choices[0].message)

    # Handle tool calls if present
    if(llm_response.choices[0].message.tool_calls):
        tools_to_run = XpanderClient.extract_tool_calls(llm_response=llm_response.model_dump())
        tool_responses = agent1.run_tools(tools_to_run)
        
        for tool_response in tool_responses:
            memory.append({
                "role": "tool", 
                "content": json.dumps(tool_response.result), 
                "tool_call_id": tool_response.tool_call_id
            })
    
    # Check for completion
    if(llm_response.choices[0].message.content):
        if "##FINAL ANSWER##" in llm_response.choices[0].message.content:
            break
            
    number_of_calls += 1

print(llm_response.choices[0].message.content)
```
</CodeGroup>

## Understanding the Code

The example above demonstrates several key concepts:

1. **Environment Setup**: We use environment variables to securely manage API keys for both xpander and OpenAI.

2. **Client Initialization**: We create instances of both the OpenAI client and xpander client, then retrieve our agent.

3. **Conversation Memory**: We maintain a list of messages that tracks the entire conversation history, including:
   - System instructions
   - User input
   - Assistant responses
   - Tool outputs

4. **Interactive Loop**: The code runs in a loop that:
   - Sends the current conversation to the LLM
   - Processes any tool calls requested by the LLM
   - Adds responses back to the conversation memory
   - Continues until the LLM indicates completion with "##FINAL ANSWER##"

5. **Tool Handling**: When the LLM requests to use a tool:
   - We extract the tool calls using `XpanderClient.extract_tool_calls()`
   - Execute them using `agent1.run_tools()`
   - Add the results back to the conversation

Each tool response includes:

- Function name used
- HTTP status code
- Response from the target system
- Generated payload
- Original tool call ID
