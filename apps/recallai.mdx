Learn how to build AI agents with Recall.ai access using xpander.ai. This integration enables intelligent meeting-aware AI assistants that can access, process, summarize, and analyze meeting recordings and transcripts automatically.

## Authentication Options

<Tabs>
  <Tab title="API Access">
    <Frame caption="Recall API access configuration">
      ![Recall API Access](/images/placeholders/recall-api.png)
    </Frame>

    For programmatic access and automation:
    1. Sign in to your Recall dashboard, there you can manage your API tokens:

        [(US) us-east-1](https://us-east-1.recall.ai/dashboard/api-keys)

        [(Pay-as-you-go) us-west-2](https://us-west-2.recall.ai/dashboard/api-keys)

        [(EU) eu-central-1](https://eu-central-1.recall.ai/dashboard/api-keys)

        [(JP) ap-northeast-1](https://ap-northeast-1.recall.ai/dashboard/api-keys)
    
    2. Create an API token and copy it to your clipboard.

  ### Let's create the Xpander.ai Recall interface!
    1. Write your interface name.
    2. Select the interface access you'd like 
    3. Select the API key authentication method
    4. Select the Integration user authentication scope
    5. Select **Custom** as the Auth type and **Authorization** as the Custom header name.
    6. Paste the token copied earlier in the API Key field using this template: `Token [YOUR_API_KEY]`
        
      **Note:** Make sure to replace `[YOUR_API_KEY]` with your actual StatusPage API key.

    <Frame>
      ![](/images/screenshots/recall_2025_2025-04-28-12-32-31.png)
    </Frame>

    Then save the interface.

    For more information about Recall's platform, [click here](https://docs.recall.ai/docs),
    and for information about Recall's API authentication, [click here](https://docs.recall.ai/reference/authentication).
