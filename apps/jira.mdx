Learn how to build AI agents with Jira access using xpander.ai. This integration enables intelligent document-aware AI assistants that can read, write, organize, and analyze issues of different projects automatically.

## Authentication Options

<Tabs>

  </Tab>
  <Tab title="API Access">
    <Frame caption="Jira API access configuration">
      ![Jira API Access](/images/placeholders/jira-api.png)
    </Frame>

    For programmatic access and automation:


    1. Sign in at
    https://id.atlassian.com/manage-profile/security/api-tokens.

    2. Click **Create API token**:
      <Frame caption="Google Drive API access configuration">
      ![Jira API Access](/images/screenshots/jira_api_token_2.png)
      </Frame>

    3. Enter a descriptive name.
      <Frame> 
      ![](/images/screenshots/jira_api_token_3.png)
      </Frame>

    4. Select the `Expires on` field:
      <Frame>
      ![](/images/screenshots/jira_api_token_4.png)
      </Frame>

	  5. Fill it with an expiry date (between 1–365 days from current date):
      <Frame>
      ![](/images/screenshots/jira_api_token_5.png)
      </Frame>

    6. Click **Create**:
      <Frame>
      ![](/images/screenshots/jira_api_token_6.png)
      </Frame>

	  7. Click **Copy to clipboard** and immediately paste the token into your script or store it somewhere secure.
      <Frame>
      ![](/images/screenshots/jira_api_token_7.png)
      </Frame>

    8. Once you copied the Token, click Done:
      <Frame>
      ![](/images/screenshots/jira_api_token_8.png)
      </Frame>

    <Warning>
    Tokens that never expire can compromise security, so Atlassian sets a one‑year expiry by default. You can choose any lifetime from 1 to 365 days when you create a token.
    </Warning>

  **Note:** After you close this window the token value cannot be retrieved. Save it right away—preferably in a password manager.
  Now we can see the tokens generated and see it was never used, but that'll change once we create the xpander.ai interface:
    <Frame>

    </Frame>
